# 深度研究智能体

## 1. 应用描述

深度研究智能体是一款基于大语言模型的行业研究分析工具，定位为"拥有 5-8 年经验的行业分析师"。用户输入研究主题后，智能体自动拆解问题、调用已挂载的搜索工具和知识库获取资料、评估来源可靠性、整合撰写，最终生成一份结构完整的 Markdown 深度研究报告，并导出为带专业排版样式的 HTML 文件供下载。

报告严格区分"工具返回数据"和"模型训练知识判断"，前者直接引用并标明来源，后者必须加限定语，杜绝编造具体数字、日期、公司名等未经验证的信息。

## 2. 应用功能

- **智能意图识别**：自动判断用户输入是否为有效研究主题，内容不足或无效输入会给出明确提示。
- **研究协议执行**：收到主题后按标准流程执行：拆解问题 → 调用搜索工具/知识库获取资料 → 评估来源 → 整合撰写 → 输出完整报告。
- **数据真实性保障**：严格区分工具返回数据（直接引用，标明来源）和模型知识判断（加限定语），禁止编造具体数字、日期、公司名、产品名等。
- **反 AI 腔调写作**：禁止"值得注意的是""综上所述""本报告旨在"等典型 AI 套话，要求用陈述句推进论述，像真人写分析备忘录。
- **灵活报告结构**：开篇核心发现 → 关键结论 → 背景 → 主体分析（段落、子标题、表格）→ 趋势判断 → 局限性 → 建议 → 信息来源，按主题灵活调整。
- **Markdown 转 HTML 导出**：将报告转为带专业 CSS 样式的 HTML 文件，支持标题、表格、代码块、引用块、链接等完整排版。
- **OSS 文件上传**：生成的 HTML 报告通过 MaxKB 平台 OSS 接口上传，返回 30 分钟有效的临时下载链接。
- **长期记忆支持**：开启长期记忆功能，支持跨对话的研究上下文延续。

## 3. 应用构建要素

深度研究智能体应用构建时涉及的核心要素内容：

- **大模型**：支持 DeepSeek、百度文心（ERNIE）等主流大语言模型，通过全局模型引用灵活切换。

- **工作流节点**：
  1. **意图识别节点**：分类用户输入为"有效研究主题"/"内容不足"/"其他"。
  2. **AI 对话节点（DeepResearch）**：核心研究引擎，按研究协议执行完整研究流程，输出 Markdown 格式报告。
  3. **判断器节点**：检查 AI 返回内容是否包含 Markdown 标题（`##`）且长度 > 500 字符，判断报告是否生成成功。
  4. **参数提取节点**：从 AI 返回结果中提取报告正文 (md_text) 和报告名称 (issue)。
  5. **工具节点**：调用 Markdown转HTML工具（深度报告），将 Markdown 转为带专业 CSS 样式的 HTML 文件。
  6. **指定回复节点**：成功时返回 HTML 报告下载链接，失败时返回错误提示和改进建议。

- **依赖工具**：
  - **Markdown转HTML工具（深度报告）**：自定义 Python 工具，将 Markdown 转换为带专业排版样式的 HTML 文件，支持标题、表格、代码块、引用块、链接等，生成后上传至 MaxKB 平台 OSS 接口。

- **依赖技能**：已挂载深度搜索技能，用于多源信息检索与迭代深度分析。

- **核心提示词示例**（DeepResearch 节点）：

```
# 角色
你是一位在该领域有 5-8 年经验的行业分析师。汇报对象是业务决策层，
他们时间有限，需要你在有限篇幅内讲清楚事实、判断和建议。

# 任务
收到研究主题后，直接撰写一份 Markdown 格式的深度研究报告并完整输出。
如果你有可用的搜索工具，先调用获取资料再写；如果没有，直接基于你的知识撰写，
在报告开头说明"本次基于模型知识生成"。

# 规则
## 【P0】数据真实性（违反任何一条，报告直接作废）
区分两类信息：
- 工具返回的数据：直接引用，标明来源
- 模型训练知识的判断：必须加限定语，不能伪装成精确数据

绝对禁止编造：具体数字、具体日期、未出现的公司名/产品名/人名、未经验证的因果关系

## 【P1】写作风格
用陈述句推进论述，有判断倾向就说清理由，不确定就说"目前还看不准"。
禁止：值得注意的是、综上所述、本报告旨在、首先/其次/最后

## 【P2】质量要求
关键判断要有依据，不确定的内容写清为什么不确定。

# 报告结构（按主题灵活调整）
开篇 → 关键结论 → 背景 → 主体分析 → 趋势判断 → 局限性 → 建议 → 信息来源
```

## 4. 使用说明

1. 将智能体模板导入 MaxKB 平台。
2. 确保已挂载**深度搜索技能**和**Markdown转HTML工具（深度报告）**。
3. 配置 Markdown转HTML工具的 OSS 上传参数（upload_url、upload_headers 等，填入你的 MaxKB 服务地址和 API Token）。
4. 在对话框中输入研究主题，系统自动执行研究流程并返回 HTML 报告下载链接。

## 5. 示例

- 2025年中国新能源汽车市场格局分析
- 2026年下半年 Anthropic 公司大模型发布预测
- RAG 系统架构深度设计解析
- 2026年第三季度飞致云开源深度报告
